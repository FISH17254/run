隐藏运动对接文档

统一下游接口 — 支持体适能 / tsn新体适能 / 云运动 / 红蜻蜓 / 位动酷跑 五大业务

📢 公告：因本学期订单增加，部分项目半路上架，实属仓促，无法统一字段，下学期会统一字段和模板。

快速开始

3 步完成对接：

步骤操作说明
1获取 uid + key联系管理员开通账号，获取 uid（用户ID）和 key（加密密钥）
2调用 getconfig获取对应服务的学校配置和单价，确认可用学校和参数
3调用 pladd传入订单数据和配置参数完成下单，保存返回的 yid 用于后续操作
后续可通过 yid 进行退款（refund）、改密码（editpass）、改状态（setstatus）等操作，均支持单个和批量

基础信息

请求地址POST {你的域名}/LearnExp/my_run/openapi.php
Content-Typeapplication/json
请求方式POST（Body 传 JSON）
URL参数?service=tsn&act=getconfig
请将 {你的域名} 替换为管理员提供的实际域名地址

认证方式

所有接口均需在 Body 中携带 uid + key（加密密钥），key 为系统后台分配的密钥，直接传入即可。

{
  "uid": 1,
  "key": "你的加密密钥"
}
key 为后台分配的加密密钥，直接填入即可。认证失败返回：{"code":-1,"msg":"身份校验失败，uid或key错误"}

服务类型（service）

service名称说明
tsn体适能带距离，按 学校+距离 计费，所有订单共用同一距离
tsn2tsn新体适能学校+跑区+距离，每个账号独立 distance，按 km×次数 计费
yyd云运动学校白名单，按次计费
hqt红蜻蜓需分配学校+校区，按次计费
wdkp位动酷跑学校+校区+距离，校区绑定距离列表，按次计费

1 getconfig POST

获取指定服务的学校配置和单价

请求

POST ?service=tsn&act=getconfig
{
  "uid": 1,
  "key": "加密密钥"
}

返回示例

{
  "code": 1,
  "msg": "查询成功",
  "data": {
    "price": 0.15,
    "service": "tsn",
    "name": "体适能",
    "schools": { ... }
  }
}

schools 结构差异

tsn（体适能）

{ "陕西科技大学": [5], "欧亚学院": [5,10,15,20] }

key=学校，value=支持的距离数组(km)

tsn2（tsn新体适能）

{ "陕西职业技术学院": { "fence": ["跑区1","跑区2"] } }

key=学校，fence=跑区列表；distance 在每条订单中单独传，不在 config 中

yyd（云运动）

["合肥工业大学", "安徽中医药大学", ...]

纯数组，学校白名单

hqt（红蜻蜓）

{ "新疆农业大学": { "fence": ["老满城","亚心"] } }

key=学校，fence=校区列表

wdkp（位动酷跑）

{ "新疆财经大学": { "本部": ["5","10"], "北校区": ["5","10"] } }

key=学校，子key=校区，value=距离数组

2 pladd POST 限流 3s

批量下单，自动扣费。不同 service 的 orders 和 config 字段有差异。

请求结构

POST ?service=tsn&act=pladd
{
  "uid": 1, "key": "加密密钥",
  "config": { ... },
  "orders": [ ... ]
}

config 参数

字段类型适用说明
sport_numint全部运动次数（1-999）
startdatestring全部开始日期 YYYY-MM-DD，不早于当天
speed_rangearray全部配速范围(min/km)，取值 3~11，如 [5, 8] 表示每公里5~8分钟
startTimeRangestring全部开跑时间段，如 "09:00~14:00"
distancefloat仅 tsn距离(km)，0.1~50，全局配置所有订单共用
run_week_dayarraytsn / tsn2跑步星期，如 ["1","2","3","4","5"]
run_week_timesintyyd / hqt / wdkp每周跑步次数（1-7）
norun_week_dayarrayyyd / hqt / wdkp（可选）休息星期，如 ["7"]，不传默认无休息日
remarkstring全部（可选）备注
蓝色行 = tsn / tsn2 使用  |  黄色行 = 仅云运动/红蜻蜓/位动酷跑使用
tsn 的 distance 在 config 全局统一；tsn2 的 distance 在每条 order 中单独传，config 中不需要 distance
tsn / tsn2 用 run_week_day 指定哪天跑；其他三个用 run_week_times + norun_week_day 控制频率

orders 每条订单字段

service必填字段可选字段说明
tsnschool, user, pass—学校需匹配 schools 配置，distance 在 config 全局统一
tsn2school, fence, distance, user, pass—school+fence 需匹配配置，distance 每人独立传，不在 config 中
yydschool, user, pass—学校需在白名单中，无 fence / distance
hqtschool, fence, distance, user, pass—school + fence 需匹配配置
wdkpschool, fence, distance, user, pass—school+fence+distance 需匹配配置，每人可不同
注意：tsn2 / hqt / wdkp 每条订单必须传 distance，各人可不同；tsn 的 distance 在 config 中全局统一设置，订单中不传

价格计算

service公式
tsn单价 × 距离(km) × 次数 × 人数（所有人共用同一距离）
tsn2各账号单独计算：∑（单价 × 该账号距离(km) × 次数）
其他单价 × 次数 × 人数

返回

{
  "code": 1,
  "msg": "下单成功",
  "data": {
    "success_count": 2,
    "total_money": 45.0,
    "orders": [
      { "yid": "AbCd1234", "school": "陕西科技大学", "user": "18888888888" }
    ]
  }
}
返回的 yid 是订单唯一标识（8~10位字母数字组合），后续退款、改密码、改状态都通过 yid 操作，请妥善保存。

获取该 uid 在指定 service 下的全部订单

{
  "uid": 1,
  "key": "加密密钥"
}

返回

{
  "code": 1,
  "data": {
    "count": 15,
    "orders": [
      {
        "id": 157, "yid": "AbCd1234",
        "school": "陕西科技大学",

        "today_status": "今日已跑",

        "start_date": "2026-03-25",
        "remark": "", "addtime": "2026-03-23 10:00:00"
      }
    ]
  }
}

返回字段说明

字段说明备注
id数据库自增ID
yid订单唯一标识8~10位字母数字
school学校
user账号
pass密码
sport_num总次数
finish_num已完成次数
today_status当前状态见下方状态值
today_distance今日跑步距离
today_speed今日配速
start_date开始日期
distance距离(km)tsn / tsn2 / hqt / wdkp
fence跑区/校区tsn2 / hqt / wdkp
remark备注
addtime下单时间

today_status 状态值

值含义能否退款/改密码
未登录初始状态，等待系统登录可以
待跑步已登录，等待开跑可以
跑步中正在执行跑步任务不可以
今日已跑今天的任务已完成可以
已完成全部次数跑完不可退款
登录失败账号密码错误可以（建议改密码）
此接口限流 60 秒一次，请勿频繁调用。建议本地缓存订单数据，仅需要时刷新。

4 refund POST 限流 3s

退款，支持单个和批量。删除订单并退还剩余次数对应金额。

请求

POST ?service=tsn&act=refund
{
  "uid": 1, "key": "加密密钥",
  "yids": ["AbCd1234"]            // 单个退款传1个
}
// 批量退款
{
  "uid": 1, "key": "加密密钥",
  "yids": ["yid1", "yid2", "yid3"]  // 批量退款传多个
}

返回

{
  "code": 1,
  "msg": "退款成功",
  "data": { "success_count": 2, "skipped": 1, "total_refund": 37.5 }
}

退款规则

条件结果
"跑步中"的订单自动跳过（计入 skipped）
已完成（剩余次数≤0）自动跳过
tsn / tsn2 退款金额单价 × 该订单距离(km) × 剩余次数
其他 退款金额单价 × 剩余次数

按学校+学号修改密码，同时重置状态为"未登录"。

请求

POST ?service=tsn&act=editpass
{
  "uid": 1, "key": "加密密钥",
  "school": "欧亚学院",
  "user": "2024010001",
  "pass": "新密码"
}
字段类型必填说明
schoolstring是学校名称
userstring是学号
passstring是新密码
该学号下有"跑步中"的订单时，整个请求会被拒绝。

返回

{ "code": 1, "msg": "修改成功", "data": { "success_count": 1 } }

6 setstatus POST

修改订单状态，支持单个和批量。

请求

POST ?service=tsn&act=setstatus
{
  "uid": 1, "key": "加密密钥",
  "yids": ["yid1", "yid2"],           // 单个或多个
  "status": "未登录"
}
字段说明
yidsyid 数组（4~20位字母数字），传1个即为单个操作
status仅支持 "未登录" 或 "待跑步"

返回

{ "code": 1, "msg": "成功修改 2 个订单状态", "data": { "success_count": 2, "skipped": 0 } }
"跑步中"的订单会自动跳过，不会报错。

数据同步机制

下游系统通过定时运行 sync.php 从上游拉取最新订单状态，自动更新本地数据库，无需调用方手动维护状态。

同步逻辑

  1. 调用上游 getorders 拉取全量订单
  2. 按 yid 匹配本地记录，批量 UPDATE 状态字段
  3. 若本地记录尚无 yid（下单时上游未返回），则按 user + addtime 回填 yid

同步字段

service同步字段
tsn / tsn2today_status, today_start_time, today_over_time, today_speed, today_distance, finish_num, name, total, order_data, remark
yydtoday_status, today_start_time, today_over_time, today_speed, today_distance, finish_num, type, order_data, name, totalkm, totalnum, remark
hqttoday_status, today_start_time, today_over_time, today_speed, today_distance, finish_num, info, total, order_data, remark
wdkptoday_status, today_start_time, today_over_time, today_speed, today_distance, finish_num, name, total, order_data, remark
同步脚本由下游服务器定时调用（如每分钟一次），调用方通过 getorders 接口读取的状态数据即为最新同步结果，无需额外处理。

错误处理

所有接口错误统一返回格式：

{ "code": -1, "msg": "错误原因描述" }

常见错误码

msg原因
缺少uid或key参数Body 未传 uid/key
身份校验失败，uid或key错误uid 或 key 不匹配
无效的服务类型service 参数错误，可选 tsn/tsn2/yyd/hqt/wdkp
操作过于频繁触发限流，等待后重试
余额不足账户余额不足以支付订单
